# Authentication System Documentation

This document details the OTP-based authentication system integrated into Arthsarthi AI Backend.
The system is passwordless, relying on email-based One-Time Passwords (OTPs) for user verification and JSON Web Tokens (JWT) for session management.

---

## 1. Overview
The authentication flow follows these primary steps:
1. **Check Email**: Verify if the email exists in the system.
2. **Request OTP**: Generate a 6-digit code and email it to the user.
3. **Verify OTP**: Validate the code and issue authentication tokens.
4. **Session Management**: maintain user state using Access (short-lived) and Refresh (long-lived) tokens.

---

## 2. Key Components

### A. Database Models (MongoDB)
- **User (`users`)**: Stores profile info (name, email, role), preferences, and metadata.
- **OTP (`otps`)**: Temporary storage for generated codes. Includes expiration (10 mins) and rate limiting (max attempts).
- **Session (`sessions`)**: Tracks active devices and refresh tokens. Allows remote logout / session revocation.

### B. Security Features
- **JWT**:
    - **Access Token**: Valid for 30 minutes. Used for API authorization.
    - **Refresh Token**: Valid for 7 days. Used to obtain new access tokens.
    - **JTI (JWT ID)**: Unique identifier for every token to support blacklisting/revocation.
- **Cookies**: Tokens are set as HTTP-only cookies (`access_token`, `refresh_token`) to prevent XSS attacks.
- **Rate Limiting**:
    - Limits OTP requests to 5 per hour per email.
    - Limits OTP verification attempts to 3 per code.

### C. Services
- **Email Service**: Uses `resend` SDK to deliver OTP emails reliably.
- **Templates**: HTML templates handle dynamic insertion of User Name and OTP Code.

## 3. Authentication Flow (Step-by-Step)

### Step 1: User Enters Email
**Frontend Action**: Call `POST /api/auth/check-email` with `{ "email": "user@example.com" }`
**Backend Logic**:
- Checks `users` collection.
- Returns `user_exists: boolean` and `requires_profile_completion: boolean`.
- **Frontend Decision**: If `user_exists` is false, prompt for First Name / Last Name (Registration Flow). If true, proceed to OTP (Login Flow).

### Step 2: Request OTP
**Frontend Action**: Call `POST /api/auth/request-otp`
- **Payload (Login)**: `{ "email": "..." }`
- **Payload (Register)**: `{ "email": "...", "first_name": "...", "last_name": "..." }`
**Backend Logic**:
- Validates rate limits.
- Generates 6-digit random code.
- Stores in `otps` (expires in 10m).
- Sends email via Resend.
- Returns `{ "success": true, "expires_in": 600 }`.

### Step 3: Verify OTP
**Frontend Action**: User enters code. Call `POST /api/auth/verify-otp` with `{ "email": "...", "otp_code": "123456" }`.
**Backend Logic**:
- Validates code matches and is not expired.
- **If New User**: Creates record in `users` collection using data stored in OTP from Step 2.
- **Session Creation**:
    - Generates Session ID.
    - Creates Access/Refresh tokens linked to Session ID.
    - Stores session in `sessions` collection.
- **Response**:
    - Sets `HttpOnly` cookies: `access_token`, `refresh_token`.
    - Returns JSON: `{ "user": { ...profile... }, "message": "Login successful" }`.

### Step 4: access Protected Resources
**Frontend Action**: Browser automatically sends `access_token` cookie with subsequent requests (e.g., to `/api/profile` or `/api/stocks`).
**Backend Logic**:
- Middleware/Dependency `get_current_user` extracts token.
- Verifies signature and expiration.
- Loads user from DB.
- Grants access.

## 4. API Endpoints Reference

### Public Endpoints
| Method | Path | Description | Payload |
|:---|:---|:---|:---|
| **POST** | `/api/auth/check-email` | Check if user is registered | `{ email }` |
| **POST** | `/api/auth/request-otp` | Trigger OTP email | `{ email, first_name*, last_name* }` |
| **POST** | `/api/auth/verify-otp` | Verify code & login | `{ email, otp_code }` |

### Protected Endpoints (Requires Login)
| Method | Path | Description |
|:---|:---|:---|
| **GET** | `/api/auth/profile` | Get current user details |
| **PUT** | `/api/auth/profile` | Update profile (name, phone) |
| **POST** | `/api/auth/logout` | Clear cookies & invalidate session |
